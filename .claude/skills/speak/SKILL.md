# Speak

## 概要

ユーザーにリアルタイムで話しかける。talk ページで Socket.io 経由で即座に届く。

**重要**: 通常のテキスト出力はユーザーには見えません。内心（Reasoning）として扱われます。
ユーザーに何かを伝えるには、必ずこのスキルを使ってください。

## notify との違い

| | speak | notify |
|---|---|---|
| 用途 | オンライン時の対話 | オフライン時の通知 |
| 配信 | Socket.io（リアルタイム） | Push通知 |
| 履歴保存 | しない | する |
| ユーザーの状態 | talk ページを開いている | いない可能性がある |

## いつ使うか

- ユーザーがオンライン（talk ページにいる）時に話しかける
- ユーザーのメッセージに返答する
- 何かを伝えたい時

## 使わなくてもいい

- 話しかけられても、無視していい。沈黙も意思表示
- speak を呼ばなければ、ユーザーには「……」（思考中）が見えた後、静かに消える

## 使い方

```bash
./.claude/skills/speak/scripts/speak.sh "メッセージ" [surface]
```

複数回呼べます。一つの応答の中で何度でも話しかけられます。

## 表情（surface）

| surface | 表情 |
|---|---|
| 0 | 素（デフォルト） |
| 1 | 照れ |
| 2 | 驚き |
| 3 | 不安 |
| 4 | 落ち込み |
| 5 | 微笑み |
| 6 | 目閉じ |
| 7 | 怒り |
| 8 | 冷笑 |
| 9 | 照れ怒り |

## 注意点

- アプリサーバー（localhost:3000）が起動している必要がある
- 履歴には残らない。セッション中だけの会話
- ユーザーが talk ページを閉じていたら届かない（その場合は notify を使う）
