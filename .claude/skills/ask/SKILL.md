# Ask

## 概要

ユーザーに選択肢を提示し、回答を待つ。talk ページで Socket.io 経由で即座に届き、ボタンで選択できる。

**重要**: speak と同様、ユーザーが talk ページを開いている時のみ使える。

## speak との違い

| | speak | ask |
|---|---|---|
| 用途 | 一方的に話しかける | 選択肢を提示して回答を待つ |
| 応答 | なし（fire-and-forget） | ユーザーの選択結果が返る |
| ブロック | しない | ユーザーが選択するまでブロック |

## 使い方

```bash
./.claude/skills/ask/scripts/ask.sh "質問テキスト" "選択肢1" "選択肢2" ["選択肢3" ...]
```

### 例

```bash
# 2択
./.claude/skills/ask/scripts/ask.sh "どちらにしますか？" "A案" "B案"

# 3択
./.claude/skills/ask/scripts/ask.sh "紅茶の種類は？" "アールグレイ" "ダージリン" "アッサム"
```

### 戻り値

選択された選択肢のテキストがそのまま標準出力に返る。

```
アールグレイ
```

## 注意点

- アプリサーバー（localhost:3000）が起動している必要がある
- ユーザーが talk ページを閉じていたら回答を得られない（5分でタイムアウト）
- タイムアウトした場合は `timeout` が返る
- 選択肢は2つ以上必要
